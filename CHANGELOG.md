# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Entries for 0.5.4 and earlier are reconstructed from git history and PR
titles; they may be less granular than releases cut under this changelog.

## [Unreleased]

## [0.5.5] — 2026-04-19

### Fixed

- `PackageManager.add_from_remote` no longer crashes with
  `AttributeError: 'dict' object has no attribute 'properties'` when pulling
  a package whose transitive dependencies are stored as dicts. The recursive
  call now resolves each sub-dependency via `remote.query()` before
  re-entering. The same fix is applied to `add_to_remote` for symmetry.

### Added

- Test suite scaffolding based on `pytest` (71 tests, <0.2s):
    - `test/conftest.py` — `tmp_edm_home` and `make_package` fixtures isolate
      `DEPMANAGER_HOME` per test.
    - Unit tests for `Props` / `Dependency` matching, version comparison,
      date parsing, and the deprecated `compiler` alias.
    - Unit tests for `Machine` introspection helpers.
    - Parser tests for `ConfigFile` (remote / packages / malformed YAML).
    - End-to-end tests for `LocalDatabase` on an on-disk tmpdir cache.
    - Non-regression tests for `add_from_remote` transitive-dep handling.
- Developer documentation under `docs/`:
    - `docs/architecture.md` — layered view, class diagram, push/pull/build
      sequence diagrams (Mermaid).
    - `docs/contributing.md` — local setup, coding conventions, release
      checklist.
    - `docs/recipe-authoring.md` — writing a build recipe.
    - `docs/extending-remotes.md` — implementing a new remote backend.
- Repository hygiene files: `CHANGELOG.md`, `CONTRIBUTING.md`, `SECURITY.md`,
  issue templates, and a pull request template under `.github/`.

### Changed

- `pytest` added as a dev dependency; `[tool.pytest.ini_options]` wires
  `testpaths = ["test"]`.
- `CLAUDE.md` updated: documents the test workflow, `docs/` layout, and the
  "diagrams must be Mermaid" convention.

## [0.5.4] — 2026-02-15

### Fixed

- Minor import bugs (#75).
- Import cleanups (#74).

## [0.5.3] — 2026-02-15

### Changed

- Switched the build system to Poetry (#70).
- Upgraded `twine` to the 6.x line (#73).

### Fixed

- Several bug fixes across the package flow (#71).
- Performance optimizations across the command pipeline (#72).

## [0.5.2] — 2026-01-05

### Added

- `--raw` output option on CMake-facing commands for cleaner scripting
  (#66).
- More verbose logging in the CMake invocation path (#68).

### Fixed

- Minor bug fixes during packaging (#67).
- Package preparation fixes in release scripts (#69).

## [0.5.1] — 2025-12-06

### Fixed

- Creation of the `~/.edm/` folder on a fresh installation (#63, #64).
- Repaired the legacy `dm_load_package` CMake function (#62).

### Changed

- Internal release-number handling (#65).

## [0.5.0] — 2025-11-18

### Added

- Packages declare their dependencies; metadata is stored and displayed
  (#56).
- `depmanager pack info` command displays package descriptions and
  metadata (#61).
- The build system both consumes and generates dependency information
  (#57).
- `depmanager load` resolves and loads dependencies automatically (#58).
- `pack pull` and `pack push` now propagate transitive dependencies (#59).
- Encrypted storage for remote-server credentials via `PasswordManager`
  (#55).
- `DEPMANAGER_HOME` environment variable overrides the default
  `~/.edm/` location (#53).

### Changed

- Improved console output across `pack ls`, `toolset ls`, `remote ls`
  (#52, #54).

### Fixed

- `pack rm` behaviour when the query matches multiple packages (#51).
- Various smaller corrections (#60).

## [0.4.2] — 2025-10-26

### Added

- Recipes can run custom code before CMake sees `CMakeLists.txt`, enabling
  packages whose sources are generated by the recipe itself (#49).

## [0.4.1] — 2025-06-20

### Fixed

- `depmanager pack add` accepts packages again after the 0.4.0 builder
  refactor (#48).
- `llvm` toolchain selection inside the builder (#48).

## [0.4.0] — 2025-06-16

### Added

- Concept of *toolsets*: named (compiler, ABI) pairs that recipes and builds
  reference explicitly (#44).
- `ls` / `rm` aliases for the verbose subcommands (#43).

### Changed

- Replaced the free-form `compiler` field by a structured `abi` value
  (`gnu` / `llvm` / `msvc`) (#46). The old `compiler` key is still read for
  backwards compatibility.
- Refactored the builder procedure for cleaner recipe lifecycle management.
- Reload of the local database now actualises entries in place instead of
  appending duplicates.

### Deprecated

- Legacy commands removed in favour of the new layout (#45).

## [0.3.4] — 2024-05-08

### Added

- Verbosity flag on the cleaning command (#41).

### Fixed

- CMake invocation command corrections (#40).
- Hot-fix on the `push` command.

## [0.3.3] — 2024-01-27

### Added

- Multi-package actions: commands accept queries that match several packages
  via `-r` (#39).

### Fixed

- Function that returns the CMake path (#38).
- Handling of single-configuration CMake generators (#37).

## [0.3.2] — 2024-01-19

### Added

- Recipe reordering: packages the project depends on build before their
  consumers (#36).

### Fixed

- CMake code corrections (#35).
- Smarter handling of already-built packages in the auto-builder (#34).

## [0.3.1] — 2024-01-16

### Added

- `RecipeBuilder` class isolating the build lifecycle (#29).
- Recursive recipe discovery (#30).
- Automatic pull step before a build (#31) and push step after a build (#32).

### Fixed

- Documentation corrections (#33).

## [0.3.0] — 2024-01-12

### Added

- `depmanager.yml` configuration file + `load` function for project-level
  package loading (#25).
- Build date and glibc tracked on every local package.
- Transitive queries across local and remote databases.
- Glibc-aware package matching in the CMake integration.
- Remote synchronization improvements: sync in both directions, delete on
  remote (#17, #18).

### Changed

- Improved locking around the shared config file (#24).
- More robust date parsing (#20, #21).

## [0.2.1] — 2023-12-31

### Fixed

- Date parsing regression from 0.2.0 (#20).

## [0.2.0] — 2023-12-12

### Added

- Support for a dedicated HTTP server remote (with optional HTTPS) alongside
  the existing FTP and folder backends.
- Remote sync: push local packages to remote, pull newer ones from remote.
- Thread-safe access to the shared config file.
- Timeout on the config-file lock to avoid stalls.
- Large-file upload support.
- System introspection improvements.

### Changed

- Migrated HTTP client from `http.client` to `requests`.

## [0.1.x] and earlier

- `0.1.4`: initial PyPI release.
- `0.1.3` and earlier: working FTP remote, folder remote, builder system,
  command refactor, API / internal / command layering, credential storage
  for remotes.
